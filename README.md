# NoA 프로젝트 소개

NoA는 멀티 에이전트 협업을 통해 안정적이고 일관된 응답을 생성하는 C 기반 런타임입니다. 단일 모델 호출이 아닌 역할 기반 에이전트(제안/검토/통합)를 조합해 응답 품질과 신뢰성을 높이며, 외부 호출자는 C ABI 또는 JSON ABI로 동일한 입력/출력 계약을 사용할 수 있습니다. 런타임은 입력 검증, 에이전트 실행 조율, 결과 합성, 오류 표준화를 담당하고, 확장 가능한 에이전트 구성과 제약 조건(동시성/타임아웃/히스토리 제한)을 통해 운영 환경의 요구사항을 충족합니다.

이 저장소는 NoA 프로젝트의 문서를 공개 배포하기 위한 **문서 전용 저장소**입니다.


## TypeScript 클라이언트
NoA 클라이언트는 TypeScript 기반 공용 런타임 레이어로, 브라우저와 Node 환경을 모두 지원하는 인터페이스와 함수 구현을 제공합니다. 세션/히스토리 관리, WASM 바인딩 호출, 요청 이벤트 리스너(코루틴) 같은 런타임 유틸리티를 포함하며, API 호출의 입력/출력 구조와 에러 표준화를 동일한 계약으로 유지합니다.

## 모듈 구성
- noa-core: C 기반 런타임과 멀티 에이전트 실행 엔진을 포함합니다.
- noa-client: TypeScript 클라이언트 및 CLI 도구를 제공합니다.
- noa-memory-kernel: 메모리 커널과 관련 테스트를 포함합니다.
- Docs: ABI, 빌드, 런타임 내부 동작 등 상세 문서를 제공합니다.

- [API.md](./API.md): 외부 호출 관점의 API 설계와 응답 구조를 설명합니다.
