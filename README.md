# NoA 프로젝트 소개

NoA는 멀티 에이전트 협업을 통해 안정적이고 일관된 응답을 생성하는 C 기반 런타임입니다. 단일 모델 호출이 아닌 역할 기반 에이전트(제안/검토/통합)를 조합해 응답 품질과 신뢰성을 높이며, 외부 호출자는 C ABI 또는 JSON ABI로 동일한 입력/출력 계약을 사용할 수 있습니다. 런타임은 입력 검증, 에이전트 실행 조율, 결과 합성, 오류 표준화를 담당하고, 확장 가능한 에이전트 구성과 제약 조건(동시성/타임아웃/히스토리 제한)을 통해 운영 환경의 요구사항을 충족합니다.

## 기획 의도: 기억/컨텍스트 매니저
LLM 기반 시스템은 컨텍스트 길이 제한과 히스토리 누적으로 인해, 대화가 길어질수록 비용이 증가하고 품질이 흔들리기 쉽습니다. NoA는 멀티 에이전트 합성뿐 아니라, “어떤 정보를 언제까지 기억할지”를 시스템 레벨에서 관리하는 컨텍스트 매니저를 핵심 구성으로 두는 것을 목표로 합니다.

- 기억은 단순 로그 저장이 아니라, 요청 단위로 회상/요약/보존을 판단해 다음 요청의 입력 컨텍스트를 최적화하는 역할입니다.
- 민감 정보(API 키 등)는 기억/출력에 포함되지 않도록 설계되어야 합니다.
- 결과적으로 사용자 경험은 유지하면서 토큰 비용과 오류율을 낮추는 것이 목적입니다.

### 기억 계층(개념)
- L1: 현재 세션에서 즉시 참조할 단기 기억
- L2: 요약된 중기 기억(세션 요약/중요 사실)
- L3: 보존 가치가 높은 장기 기억(규칙/선호/핵심 맥락)

### 컨텍스트 출력(개념)
- 입력(메타/대화/페르소나/상태)을 받아 단일 문자열 컨텍스트로 병합해 런타임에 제공
- 상태 갱신과 보존 판단(유지/삭제)을 함께 산출

## TypeScript 클라이언트
NoA 클라이언트는 TypeScript 기반 공용 런타임 레이어로, 브라우저와 Node 환경을 모두 지원하는 인터페이스와 함수 구현을 제공합니다. 세션/히스토리 관리, WASM 바인딩 호출, 요청 이벤트 리스너(코루틴) 같은 런타임 유틸리티를 포함하며, API 호출의 입력/출력 구조와 에러 표준화를 동일한 계약으로 유지합니다.

## 모듈 구성
- noa-core: C 기반 런타임과 멀티 에이전트 실행 엔진을 포함합니다.
- noa-client: TypeScript 클라이언트 및 CLI 도구를 제공합니다.
- noa-memory-kernel: 메모리 커널과 관련 테스트를 포함합니다.
- Docs: ABI, 빌드, 런타임 내부 동작 등 상세 문서를 제공합니다.
- dist/wasm: WebAssembly 빌드 산출물을 포함합니다.
- scripts: 빌드 및 자동화 스크립트를 제공합니다.
- test: 런타임 관련 테스트를 포함합니다.
