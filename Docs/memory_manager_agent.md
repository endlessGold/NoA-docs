# memory_manager_agent

# 1. 기능 개요
본 문서는 NoA 프로젝트에서 포함되어있는 `noa-memory-kernel` 모듈을, 멀티 에이전트 중 **기억 매니저 역할을 담당하는 AI 에이전트**의 기능과 입력/출력 계약을 정의한다.

# 2. 도메인 맥락
- NoA는 Proposer/Reviewer/Fusion/Reflection 등 다층 에이전트 협업 구조를 가진다.
- 기억 매니저 에이전트는 NoA 파이프라인의 컨텍스트 최적화를 담당하며, L1/L2/L3 계층 기억을 관리한다.
- 기억 매니저는 NoARequestMeta, NoAMessageList, NoAPersonaList와 같은 NoA ABI 입력을 바탕으로 메모리 회상/요약/보존 판단을 수행한다.

가정:
- 기억 매니저는 **추론용 컨텍스트를 생성·제공**하는 역할로 정의되며, 최종 응답 생성은 다른 에이전트가 수행한다.

# 3. 기능 요구사항
1) NoA 요청 단위마다 기억 매니저는 현재 대화와 과거 기억을 종합해 컨텍스트를 반환해야 한다.
2) L1이 가득 찼을 때 요약 및 플러시를 수행해야 한다.
3) 중요도/최근성/유사도 기반 우선순위를 계산해 기억 유지 여부를 결정해야 한다.
4) 모호한 경우 외부 AI 판단 콜백을 통해 보존 여부를 결정해야 한다.
5) 기억 매니저는 에이전트 역할 정의에 따라 특정 모델이 담당하도록 설정 가능해야 한다.
6) NoA 파이프라인에서 컨텍스트 전달 흐름이 명시적으로 정의되어야 한다.
7) 기억 상태는 NoSQL 문서 스냅샷으로 저장·복원되어야 한다.
8) 기억 노드는 활성화·연결 구조로 관리되어야 한다.

# 4. 입력 정의
## 4.1 필수 입력
- **NoARequestMeta**: 요청 메타 정보 (prompt, system_prompt, model, base_url, mix_strategy)
- **NoAMessageList**: 현재 대화 히스토리

## 4.2 선택 입력
- **NoAPersonaList**: 기억 매니저 전용 페르소나
- **NoaMemManagerState**: L1/L2/L3 + 노드 상태 스냅샷
- **RetentionConfig**: w1, w2, w3 가중치 및 임계값

가정:
- NoaMemManagerState는 NoSQL 문서 저장소 또는 런타임 내부에서 주입 가능해야 한다.

# 5. 출력 정의
## 5.1 정상 출력
- **MergedContext**: L1/L2/L3 기반으로 병합된 단일 문자열 컨텍스트
- **UpdatedState**: L1/L2/L3 상태 갱신 결과
- **RetentionDecisionList**: 기억 항목별 유지/삭제 판단 및 점수

## 5.2 오류 출력
- **Error**: 입력 검증 실패, 요약 실패, 메모리 부족 등 원인 식별 가능한 오류 메시지

가정:
- 오류 발생 시에도 `MergedContext`는 빈 문자열로 반환 가능해야 한다.

# 6. 비기능 요구사항
1) **성능**: 대화 히스토리 길이가 증가해도 선형 또는 준선형 시간 복잡도를 유지해야 한다.
2) **메모리 제한**: L1/L2/L3 최대 용량은 상한을 가져야 하며 초과 시 안전하게 플러시해야 한다.
3) **보안**: API 키 및 민감 정보는 메모리 매니저 출력에 포함하지 않는다.
4) **이식성**: C ABI 및 JSON ABI 기반 호출과 호환되어야 한다.
5) **신뢰성**: 입력이 유효하지 않아도 시스템 전체가 중단되지 않아야 한다.

# 7. 경계 조건 및 제약
1) 입력 prompt가 비어 있으면 요약/회상 대신 오류를 반환해야 한다.
2) NoaMemManagerState가 없을 경우 빈 상태로 초기화하고 처리해야 한다.
3) mix_strategy가 존재하더라도 기억 매니저 역할의 입력/출력은 독립적으로 유지되어야 한다.
4) L1/L2/L3의 용량 초과 시 손실 정책(요약/삭제 우선순위)을 명확히 정의해야 한다.
5) 컨텍스트 길이 상한을 초과할 경우 잘라내기 또는 축약 정책을 적용해야 한다.

# 8. 미정 사항
1) 기억 매니저 에이전트가 NoA 파이프라인의 어느 시점에서 실행되는가?
2) MemoryManagerState의 저장 위치와 수명은 어떻게 정의되는가?
3) w1/w2/w3 기본값 및 조정 정책은 무엇인가?
4) L1/L2/L3의 구체적인 최대 용량 기준은 무엇인가?
5) 요약 실패 시 재시도 정책이 필요한가?
6) 컨텍스트 길이 상한 초과 시 우선 삭제 기준은 무엇인가?
