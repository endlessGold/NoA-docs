# 1. 기능 개요
build-wasm.ps1은 NoA C 소스를 WebAssembly로 빌드하는 스크립트입니다.

# 2. 도메인 맥락
WASM 산출물은 NoA 런타임을 웹/노드 환경에서 사용할 수 있게 합니다.

# 3. 기능 요구사항
1) 소스/헤더 경로를 확인해야 한다.
2) emcc가 존재하면 직접 빌드해야 한다.
3) emcc가 없으면 WSL에서 빌드해야 한다.
4) 산출물을 dist/wasm에 생성해야 한다.

# 4. 입력 정의
- OutDir: 출력 디렉터리 경로(선택)

# 5. 출력 정의
- noa.js
- noa.wasm

# 6. 비기능 요구사항
1) 빌드 실패 시 오류를 반환해야 한다.
2) 경로 변환은 플랫폼 규칙을 준수해야 한다.

# 7. 경계 조건 및 제약
1) 소스가 없으면 즉시 실패해야 한다.
2) OutDir이 없으면 기본 경로를 사용해야 한다.

# 8. 미정 사항
1) emcc 버전 호환 범위
2) 빌드 플래그 확장 정책

[논리적 검증 결과]
- 누락 항목: emcc 버전 정책, 플래그 확장 정책
- 충돌 항목: 없음
- 모호한 항목: 기본 OutDir 고정 여부

## 예시

```javascript
const out_dir = "dist/wasm";
run_script("build-wasm.ps1", out_dir);
```
