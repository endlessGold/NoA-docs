# 1. 기능 개요
NoAResponse는 실행 결과 텍스트와 오류 메시지를 반환하는 구조체입니다.

# 2. 도메인 맥락
NoA 실행 결과를 호출자에게 전달하는 표준 반환 형식입니다.

# 3. 기능 요구사항
1) text와 error 필드를 포함해야 한다.
2) 오류 발생 시 error에 메시지를 설정해야 한다.
3) 합성 전략이 적용된 경우 text는 합성 결과를 포함해야 한다.

# 4. 입력 정의
- text: string
- error: string

# 5. 출력 정의
- 성공 시 text는 응답 텍스트를 포함한다.
- 실패 시 error는 오류 메시지를 포함한다.

# 6. 비기능 요구사항
1) 문자열은 UTF-8 인코딩이어야 한다.

# 7. 경계 조건 및 제약
1) text와 error 중 어느 하나는 비어 있을 수 있다.
2) 오류 분류는 호출자가 해석할 수 있는 형태여야 한다.

# 8. 미정 사항
1) error 문자열 표준 포맷
2) text 길이 제한 정책
3) 일부 모델 실패 시 text/error 설정 규칙

가정:
- 합성 전략 사용 시에도 반환 구조는 동일하다.

[논리적 검증 결과]
- 누락 항목: error 포맷, text 길이 제한, 부분 실패 처리 규칙
- 충돌 항목: 없음
- 모호한 항목: 성공 시 error 처리 규칙, 합성 실패의 오류 분류

## 예시

```javascript
const response = { text: "합성 결과", error: "" };
```
